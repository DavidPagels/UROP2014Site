Graph Algorithm Visualization Program

Created by David Pagels for UROP 2014

Download the runnable .jar of this project

About

This program allows computer scientists learning graph theory to be able to watch how the complex algorithms unfold while also providing experienced computer scientists a tool to help develop novel graph algorithms. It does this by initially aiding the user on the use of the new language with a choice of fundamental graph theory algorithm examples. The program allows users to construct their own unique graphs in an intuitive way. These graphs may exhibit any combination of basic graph attributes. During the construction of a new graph, the user is prompted to uniquely identify each vertex. The user is also prompted on weighted graphs to enter in edge weights. At any point during the use of the program on the main screen, the user may zoom in or out and move vertices around using the arrow tool, which allows the user to situate the vertices in a way that is useful.

The other facet to this program is the new language (visLang) created to interact with the user-created graphs. This new language has several fundamental components such as if statements, while statements, for loops, functions, variables, and variable operations. Once the user creates their graph and writes their algorithm, the user can then choose how they wish to step through the algorithm. Independent of the user’s choice, their algorithm is checked for errors before attempting to run it. The options for executing through the algorithm are ‘Step’ping through the algorithm, ‘Run’ning the algorithm which automatically takes steps at a controllable interval, and an ‘Instant’ option which executes the steps as fast as the program can parse the algorithm. Also, while the user is stepping through or running their algorithm, helpful prompts are displayed below the graph they created. Some of this useful information displayed includes stating the relevant vertex, showing the value of an expression (mathematical or conditional), and telling the user a value being assigned to a certain vertices’ attribute.

There are several other notable features incorporated in this program. The user can resize the graph, move vertices, and select vertex and edge attributes they wish to view at any time (including during execution of their algorithm). The user also has the option to save and open graphs. The save feature saves whether or not the graph is weighted or directed, the position of every vertex, that vertex’s attribute values, the adjacent edges and their attributes, and the algorithm entered in the algorithm panel.

To get started, download and run the .jar file. On the 'Graph Specs' screen, press 'Create Graph' to get to the main screen. Once on the main 'GAVP' screen, an examples drop down menu can be found in the menu bar along the top. Select any of the examples to see appropriate syntax of the visLang language and to play around with the interface in order to get a better understanding of how it works.

The examples drop down menu.

visLang

visLang is a language created to interact with the visual representation of the graph constructed by the user. visLang is case and space sensitive language with all keywords being lowercase. Most, if not all statements begin with a keyword such as set, declare, add, etc.


Setting Variables


set <vartype> <varname> [<attribute>] <to|=> <expression>

vartype: int, string, vertex, edge

varname: user defined variable names

attribute: needed for vertices and edges to specify what attribute to change. (see below)

to and =: may be used interchangeably

expression:the expression is evaluated and assigned to the variable. (see below)



Declaring Variables

Variables can also be implicitly declared by setting an undeclared variable.


declare <vartype> <varname>

vartype: int, string

varname: user defined variable names



Adding Variables to Lists:


add <vartype> to <listname> = <var | varList>

vartype: variable type (int, string, vertex)

listname: name of the list to add to

var: integer, string, vertex, Graph.vertices (all of the vertices in the graph)

varList: <var>, <var> , <var> , ...



If Statements

Be certain to have a space between all tokens.


if <conditional>
    <statement>
end if

if <conditional>
    <statement>
else
    <statement>
end if

See below for more about conditional statements.


While Statements


while <conditional>
    <statement>
end while

Calling Functions

Functions may be called recursively!


call <functionName> ( <varname> , …)

functionName: user defined function name

varname: user defined variable name



Functions


function <functionName> ( <vartype> <varname> , ... )
    <statement>
end function


functionName: user defined function name

vartype : int, string, vertex

varname: user defined variable name



Return Statements:

Returning values not yet supported


return

For Loops


for each <vartype> <varname> (adjacent to | in) <varname (if “adjacent to”) or  listname ( if “in”)>
    <statement>
end for


vartype: vertex

varname: user defined variable name

listname: user defined list variable name



Conditional

Conditional statements are parsed by javascript. Currently expressions do not change the value of the variables within the conditional statement.


Special conditional statements
<vertexID> isIn (<vertexList> | Graph.vertices)

- Returns true if the vertex was found ( In the list | In the graph vertices)


<vertexID> dist

- Returns the distance value in the vertex (Double)


<vertexID1> to <vertexID2> weight

- Returns the edge weight stored in edge vertexID1 to vertexID2


<vertexList> isEmpty

- Returns true if the vertex list is empty



Expressions:

Mathematical expressions using user defined variables, integers, etc.


Special Expressions:
dequeue <vertexListName>

- Returns the first element in the list.


<listName> extract (min | max) (dist | time)

- Returns the vertex with the min/max time/dist in the list



Mark:

Use this function to mark a certain vertex (such as the starting vertex in the algorithm).


mark vertex <varName>

varName: Name of the vertex.



Special Functions


clearGraph: resets attributes of all vertices and edges to default values;




Graph Component Information


Vertex Attributes


ID:

Holds the unique string ID of itself.

Type: String

Default Value: Set by user.

predecessor:

Holds the unique string ID of another node.

Type: String

Default Value: “”

color/hue/saturation:

Color can be set to distinguish a vertex

Type: Color

Default Value: null

visited:

Marks a vertex as visited with a cyan ring

Type: boolean

Default Value: false

distance:

Distance unrelated to adjacent edge weights

Type: double

Default Value: 0.0

time:

A second variable attribute for use in algorithms.

Type: double

Default Value: 0.0



Edge Attributes


toVertex:

Holds the unique string ID of the vertex at which the edge terminates

Type: String

edgeWeight:

Holds the user-assigned edge weight.

Type: Double

color/hue/saturation:

Color can be set to distinguish an edge

Type: Color

Default Value: null



Note about Colors:


Hue values can be set between 0 and 359, Saturation can be set between 0 and 99. Colors (not hue or saturation) are limited to the Java defaults:

  • black
  • blue
  • cyan
  • darkGray
  • gray
  • green
  • lightGray
  • magenta
  • orange
  • pink
  • red
  • white
  • yellow
